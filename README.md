# carpal-docs

Открытая документация проекта [CarPal](https://car-pal-tau.vercel.app/) —
peer-to-peer marketplace для аренды машин иностранным туристам в
Кыргызстане.

## Зачем отдельный репо

Стратегические доки живут отдельно от кода по двум причинам:

1. **Шерится одной ссылкой.** Инвестору, советнику, новому соавтору — даю
   ссылку на `github.com/<...>/carpal-docs` и человек видит всю модель,
   решения и контекст без клонирования всей кодовой базы.
2. **Open startup сигнал.** Когда репо публичный (а его и хочется
   опубликовать со временем) — это сильный сигнал серьёзности и
   прозрачности. Закрытые стартапы пишут доки в Notion, открытые — в Git.

Инженерные доки (как устроен код, схема данных, API-контракты) остаются
в основном репо `carpal/`, потому что они ходят рядом с кодом и
ломаются вместе с ним.

## Структура

| Папка | Что внутри |
|---|---|
| `strategy/` | Монетизация, take rates, лог стратегических решений (ADR-lite), портреты сегментов |

Когда добавляются новые категории — режем подпапки от существующих,
не предзаготавливаем пустые. См. `strategy/README.md`.

## Конвенции

- **Русский.** Это рабочий язык основателя для стратегического мышления.
  Когда документ дозревает до публикации для англоязычных инвесторов —
  переводится отдельным файлом с суффиксом `.en.md`.
- **Числа цитируют источник или помечены `[TBD]` / `[ОЦЕНКА]`.** Никаких
  выдуманных метрик.
- **Зачёркивание для отжитых утверждений** вместо удаления — мы хотим
  видеть, во что верили раньше.
- **Один PR — одно решение.** Каждое стратегическое изменение — отдельный
  PR с обсуждением в описании. Это работает как лог через GitHub-историю.

## Связанные репозитории

- `car-pal-tau` (Vercel) — публичный фронт
- `carpal-marketing` — пре-лонч маркетинг и сбор заявок (внутри `carpal/marketing/`)
